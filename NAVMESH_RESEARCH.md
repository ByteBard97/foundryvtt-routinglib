# Navigation Mesh Integration Research

This document tracks the research and planning for replacing the current grid-based A* pathfinding with a more advanced navigation mesh (navmesh) system, leveraging the `three-pathfinding` library.

## 1. High-Level Goal

Our objective is to dynamically generate a 2D navigation mesh from the wall data within a Foundry VTT scene. This will enable more accurate and natural pathfinding for tokens of all sizes.

## 2. The Core Challenge: A Multi-Stage Geometry Problem

The research confirms that this is not a simple task for a single library. We must build a pipeline to transform the data from Foundry VTT's format to what `three-pathfinding` requires.

- **Foundry VTT Input:** Defines environments by what is **impassable** (an array of wall line segments: `[{x1, y1, x2, y2}, ...]`).
- **`three-pathfinding` Input:** Requires a definition of what is **walkable** (a triangulated mesh with vertices and indexed faces).

The pipeline must solve the "inverse problem" of deriving walkable areas from a set of restrictive obstacles.

## 3. Library & Algorithm Evaluation

### Step 1: Pre-processing and Polygonization (The Hardest Part)

This step converts raw, potentially messy wall segments into clean, enclosed polygons that represent walkable space.

- **Top Candidate: `JSTS (JavaScript Topology Suite)`**
    - **Description:** A powerful port of the robust Java Topology Suite. It is the definitive tool for this stage.
    - **Key Features:**
        - **`GeometrySnapper`:** Cleans input data by snapping nearly-touching wall endpoints together, fixing small gaps.
        - **`Polygonizer`:** The core algorithm that takes a network of lines and correctly derives the enclosed polygons, automatically identifying and creating holes.
    - **Verdict:** **Essential.** JSTS is the cornerstone of a robust solution. It directly solves the most complex part of our problem.

### Step 2: Triangulation

Once we have clean polygons with holes, we need to triangulate them.

- **Top Candidate: `poly2tri.js`**
    - **Description:** A JavaScript library for Constrained Delaunay Triangulation (CDT).
    - **Why it's a good fit:**
        - **High-Quality Triangles:** CDT produces "fat" triangles, which are numerically stable and ideal for pathfinding and funneling algorithms.
        - **Explicit Hole Support:** Has a clear API for passing in the hole data generated by JSTS.
    - **License:** BSD-3-Clause (Permissive)
    - **Verdict:** **Recommended Triangulator.** Its focus on triangle quality makes it superior for navmesh generation.

- **Alternative: `earcut`**
    - **Description:** Extremely fast ear-clipping triangulation.
    - **Verdict:** A viable alternative if raw speed is the absolute priority, but the potential for "sliver" triangles makes `poly2tri.js` a safer bet for pathfinding quality.

## 4. The New Implementation Plan: The JSTS Pipeline

Our plan is now a robust, multi-stage process, managed by a new `NavmeshBuilder` class.

1.  **Get Scene Boundaries:**
    - Get the scene's dimensions (`canvas.dimensions`). These form the outer boundary of our walkable space.
    - Get all wall line segments from `canvas.walls`.

2.  **Pre-process with JSTS:**
    - Combine scene boundary lines and wall lines into a single geometry collection.
    - Use `JSTS.operation.overlay.snap.GeometrySnapper` to clean up the geometry, closing small gaps between wall endpoints to ensure the data is "noded" correctly.

3.  **Polygonize with JSTS:**
    - Feed the cleaned line network into `JSTS.operation.polygonize.Polygonizer`.
    - This will output one or more `Polygon` objects. Each polygon will have an "exterior shell" (the walkable area) and potentially one or more "interior rings" (the holes).

4.  **Triangulate with `poly2tri.js`:**
    - For each polygon from JSTS:
        - Create a `poly2tri.SweepContext` with the exterior shell.
        - Add the interior rings to the context using `addHole()`.
        - Run the triangulation.

5.  **Format for `three-pathfinding`:**
    - Write a converter function that takes the output from `poly2tri.js` (a list of triangle objects).
    - This function will create two flat arrays:
        - `vertices`: `[x1, y1, 0, x2, y2, 0, ...]`
        - `faces`: `[idx1, idx2, idx3, ...]`
    - This is the final data structure needed by `three-pathfinding`.

6.  **Create the Zone:**
    - Pass the formatted vertices and faces to `Pathfinding.createZone()` to get the final, usable navmesh.

## 5. Next Steps

1.  **Install Dependencies:** Add `jsts`, `poly2tri.js`, and `three-pathfinding` to the project's dependencies.
2.  **Implement `NavmeshBuilder`:** Create a new file, `js/navmesh_builder.js`, and begin implementing the 6-step pipeline described above.
3.  **Visual Debugger:** Create a tool to render the output of each stage (cleaned lines, polygonized shapes, final triangulation) onto the canvas. This will be invaluable for troubleshooting.
