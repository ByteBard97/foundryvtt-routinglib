# 🧭 Navmesh for routinglib: Project Overview

This document provides a central reference for the project to replace routinglib's grid-based pathfinding with a modern navigation mesh (navmesh) system.

---

## 🎯 **Primary Objective**

The goal is to replace the current A* grid-based pathfinding with a high-fidelity navmesh system using the `three-pathfinding` library. This will provide more accurate, performant, and realistic pathfinding by operating on the scene's true geometry instead of a grid approximation.

## 📚 **Core Libraries & Technologies**

-   **`three-pathfinding`**: The core library for pathfinding on the generated navmesh. A local copy is included in the project.
-   **`JSTS (JavaScript Topology Suite)`**: Essential for robustly processing wall geometry, cleaning it, and calculating the "walkable" polygons from restrictive wall segments.
-   **`poly2tri.js`**: Used for Constrained Delaunay Triangulation (CDT) to create a high-quality mesh of triangles from the polygons generated by JSTS. This triangle quality is ideal for pathfinding.
-   **`Foundry VTT v13 (Node.js)`**: The target development environment.

## 🛠️ **Core Implementation Pipeline: The "Geometry Factory"**

The heart of this project is the `NavmeshBuilder` class, which orchestrates a multi-stage pipeline:

1.  **Collect Geometry**: Gather wall and scene boundary line segments from the Foundry VTT canvas.
2.  **Polygonize with JSTS**: Clean the geometry and calculate the enclosed "walkable" polygons, correctly identifying interior holes.
3.  **Triangulate with `poly2tri.js`**: Convert the walkable polygons into a high-quality triangulated mesh.
4.  **Format for `three-pathfinding`**: Convert the triangle data into the specific vertex and indexed face format required by the library.
5.  **Create Zone**: Load the final data into `three-pathfinding` to create a usable pathfinding zone.
